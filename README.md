## üèûÔ∏è Microbialites Database System
üîó **Repository:** [*https://github.com/ranglall/Microbialites*](https://github.com/ranglall/Microbialites)

I chose to include this project because it involved working on a real system for a real department at St. Thomas. In collaboration with the Geology Department, we replaced an existing Microsoft Access database with a SQL database and a Python-based user interface. The goal was to make the data easier to manage, update, and maintain over time. One of the main challenges of this project was learning to work with geology-specific terminology and data structures that I had never encountered before, which required frequent communication and clarification with the department like a real client relationship.

This project gave me hands-on experience with database design, data migration, and building software for users who are not programmers. It was especially valuable because it required thinking beyond just getting the code to run, and instead focusing on usability, permission levels, and long-term reliability in a real academic setting.

## üìç St. Paul Crime Map Single-Page Application
üîó **Repository:** [*https://github.com/ranglall/webdev-crimespa*](https://github.com/ranglall/webdev-crimespa)

This project is a single-page web application that allows users to explore crime data across St. Paul neighborhoods using an interactive map. It was built with Vue.js and integrates several APIs, including Leaflet for mapping, Nominatim for location search, and a custom REST API for crime data.

I selected this project because it shows my ability to build a full web application that connects frontend and backend components. It also demonstrates how data can be presented in a way that is more accessible and useful for users, rather than just displaying raw tables or numbers.
